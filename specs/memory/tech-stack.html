Princípios de stack

  • Sem build step — todos os jogos web são index.html + JS; abrem direto no browser.
  • Sem assets externos em runtime — sprites, sons e modelos 3D são procedurais/programáticos; zero dependências de load externo que possam falhar.
  • Vendor local commitado em vendor/ — libs de terceiros baixadas uma vez e versionadas no repo; a infra de testes exige operação offline (NFR-02 do release testing-infra-v1).
  • Zero TypeScript nos jogos web — JS puro para simplicidade e sem compilação.

Padrão de deploy

  • Jogos web (degraus 1 e 2): GitHub Pages (branch gh-pages, cada jogo em subpasta).
  • Jogos Godot 4 (degrau 3): binário Linux x64 via godot --export-release; distribuído via README link. Operator-only local runs em aero-fighters-v2/ (sem GitHub Releases, itch.io ou Steam).
  • Jogos UE5 (degrau 4, futuro): executável buildado e linkado no README. Bloqueado por hardware atual.
