Conceito: objetivos de produto e critérios de qualidade · Última atualização: 2026-05-17

Objetivos

  1. Criar jogos realmente jogáveis para Tauan — não demos, não protótipos descartáveis.
  2. Capturar a experiência sensorial de jogos de referência (Chrome Dino, Aero Fighters Assault N64) e adaptá-la ao público-alvo.
  3. Todo jogo deve passar em testes Playwright automatizados antes de ser entregue ao operador para revisão final.
  4. Publicar jogos jogáveis online (GitHub Pages) quando atingem maturidade visível ao jogador.

Critérios de qualidade não-negociáveis

  • Imediatamente jogável — sem tela de loading visível, sem erros no console, controles simples descobertos em segundos.
  • Smoke Playwright passando — todo jogo tem ao menos um teste smoke em tests/<jogo>/ que valida boot sem console errors.
  • Operação offline — vendor local de libs, sem dependência de CDN ou rede em runtime (NFR-02 do release testing-infra-v1).
  • Sem regressão entre releases — toda nova release roda a suíte completa (npm run test:aero:qa ou equivalente do jogo afetado) antes de fechar.

Ciclo de release

Toda evolução de jogo entra como uma release em specs/releases/<release-id>/ com tripleto SPEC + PLAN + TASKS aprovado pelo operador. Releases não passam por CLOSURE até que a QA suite do jogo afetado passe e que TASKS.md tenha critérios de conclusão marcados.
